# Отчет по доработке TaskManager

## Цель
Добавить трассировку основных операций `Add`, `Remove`, `List` с фиксацией:
- начала и конца операции;
- времени выполнения;
- результата (`успех`, `предупреждение`, `ошибка`).

## Что реализовано
1. Добавлен контекст трассировки `trace_operation` в `task_manager.py`.
2. Операции `add_task`, `remove_task`, `list_tasks` обернуты в трассировку.
3. Логи пишутся в консоль и в файл `task_manager.log`.
4. Для бизнес-результатов введены записи вида `RESULT | operation=... | status=...`.

## Формат трассировочных событий
- `TRACE START | operation=<Add|Remove|List>`
- `TRACE END | operation=<...> | result=<SUCCESS|ERROR> | duration_ms=<...>`
- `RESULT | operation=<...> | status=<SUCCESS|WARNING|ERROR> | ...`

## Проверка
Запуск выполнен командой:

```bash
python main.py
```

Наблюдаемые результаты:
- `Add`: успешное добавление задач и предупреждение для пустой строки;
- `Remove`: успешное удаление существующей задачи и предупреждение при отсутствии задачи;
- `List`: успешный вывод текущего списка задач;
- для каждой операции присутствуют `TRACE START` и `TRACE END` с `duration_ms`.

## Вывод
Трассировка для ключевых операций добавлена и работает корректно.  
Логи позволяют пошагово восстановить ход выполнения и итог каждой операции.
