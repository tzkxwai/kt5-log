# Отчет по заданию: централизованная обработка исключений

## Что требовалось
1. Добавить централизованную обработку исключений в `TaskManager`.
2. Логировать все перехваченные исключения с:
   - текстом ошибки (`Exception.Message`);
   - стеком вызовов (`Exception.StackTrace`);
   - контекстом операции (`Add`, `Remove`, `List`, и т.д.);
   - уровнем ошибки (`Error`, `Fatal`).
3. Реализовать (опционально) механизм оповещения.

## Что сделано
- Добавлен единый метод-обертка `TaskManager._execute`, через который выполняются основные операции.
- Добавлен `ExceptionHandler`, который централизованно логирует:
  - `message`;
  - `stack_trace`;
  - `operation`;
  - `level`.
- Добавлен `AlertService`:
  - выводит в консоль сообщение `!!! Ошибка !!!`;
  - пишет событие оповещения в лог;
  - при `CRITICAL` пишет запись о внешней отправке (`ALERT_EXTERNAL` как точка интеграции).

## Проверка
Запуск:

```bash
python main.py
```

Проверенные кейсы:
- `Add` с валидной задачей — успех;
- `Add` с пустым текстом — перехват исключения, логирование и оповещение;
- `Remove` для несуществующей задачи — перехват исключения, логирование и оповещение;
- отдельный `CRITICAL`-сценарий (`Notification`) — логирование уровня `CRITICAL` и запись о внешнем канале.

## Результат
Требования задания выполнены: централизованный перехват исключений внедрен, логирование расширено, механизм оповещения присутствует, отчет оформлен.
